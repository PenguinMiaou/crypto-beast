> 目标：解决策略同质性、缺乏自适应风控、无回测验证、无 ML 增强的四大短板

## 1. 背景与动机

v1.6.0 修复了 18 项审查问题后，系统基础架构已健壮。但实盘表现暴露了更深层问题：

- **3/17-19 胜率 75-100%（+$35），3/20-21 暴跌到 17-25%（-$25）**
- 根因：5 个策略中 3 个是趋势跟随型（trend_follower, momentum, breakout），市场反转时集体失败
- 风控没有基于近期表现自适应调整，连败时未缩仓
- Kelly Criterion 已计算但未接入仓位决策
- 没有回测框架验证策略有效性
- Regime 检测依赖滞后的 ADX 规则

### 研究基础

本设计基于 4 位研究员的独立调研（策略、回测、ML、风控）和全网开源社区调研（Freqtrade/FreqST/NostalgiaForInfinity/strat.ninja），团队讨论后达成共识。

### 关键共识

- **不做 Grid Trading**：$200 太小、消耗 algo order 配额、与信号驱动架构不兼容
- **不迁移 Freqtrade/Jesse**：现有 BacktestLab + optuna 增强即可
- **不做深度学习/RL**：数据量不足、$200 试错成本太高
- **借鉴 FreqST 社区**：IchiV1 策略逻辑值得移植

### 3.1 #1 Adaptive Risk — 近期表现驱动的动态风控

**文件**：`defense/risk_manager.py`

**问题**：3/19 连续亏损后系统仍以正常仓位继续开仓，没有"刹车"机制。

**前提改动**：`RiskManager.__init__` 需要新增 `db: Database` 参数（当前只有 `config: Config`）。所有创建 `RiskManager(config)` 的调用点需更新为 `RiskManager(config, db)`。涉及 `crypto_system.py` 和测试文件。

**设计**：在 `validate()` 开头加入 adaptive 检查，基于最近 10 笔交易的表现动态调整仓位：

```python
class AdaptiveRiskState:
    """Track recent performance for adaptive position sizing."""

    def __init__(self, db: Database, lookback: int = 10, cooldown_hours: int = 2):
        self._db = db
        self._lookback = lookback
        self._cooldown_hours = cooldown_hours
        self._cache_time: Optional[datetime] = None
        self._cached_scale: float = 1.0
        self._cooldown_until: Optional[datetime] = None  # 冷静期结束时间

    def get_scale_factor(self) -> float:
        """Calculate position scale factor based on recent performance.

        Returns 0.25-1.5 multiplier for position size.
        Caches result for 60 seconds to avoid DB spam.
        """
        now = datetime.now(timezone.utc)

        # 冷静期检查：胜率太低触发暂停，超时后以 0.5x 恢复
        if self._cooldown_until is not None:
            if now < self._cooldown_until:
                self._cached_scale = 0.0
                return 0.0
            else:
                # 冷静期结束，以 0.5x 恢复
                self._cooldown_until = None
                self._cached_scale = 0.5
                self._cache_time = now
                return 0.5

        if self._cache_time and (now - self._cache_time).total_seconds() < 60:
            return self._cached_scale

        rows = self._db.execute(
            "SELECT pnl FROM trades WHERE status='CLOSED' AND pnl IS NOT NULL "
            "ORDER BY exit_time DESC LIMIT ?", (self._lookback,)
        ).fetchall()

        if len(rows) < 3:
            self._cached_scale = 1.0
            self._cache_time = now
            return 1.0

        pnls = [r[0] for r in rows]
        wins = sum(1 for p in pnls if p > 0)
        win_rate = wins / len(pnls)

        # Count consecutive losses from most recent
        consecutive_losses = 0
        for p in pnls:
            if p <= 0:
                consecutive_losses += 1
            else:
                break

        # Scale factor rules
        if win_rate < 0.30:
            # 进入冷静期
            self._cooldown_until = now + timedelta(hours=self._cooldown_hours)
            scale = 0.0
            logger.warning(f"Adaptive cooldown: win_rate {win_rate:.0%} < 30%, pausing {self._cooldown_hours}h")
        elif consecutive_losses >= 5:
            scale = 0.25  # 连败≥5: 仓位25%
        elif consecutive_losses >= 3:
            scale = 0.50  # 连败≥3: 仓位50%
        elif win_rate > 0.70 and consecutive_losses == 0:
            scale = 1.3   # 高胜率+无连败: 轻度扩仓
        else:
            scale = 1.0   # 正常

        self._cached_scale = scale
        self._cache_time = now
        return scale

    def get_min_confidence_boost(self) -> float:
        """Additional min_confidence requirement during losing streaks."""
        scale = self.get_scale_factor()
        if scale <= 0.25:
            return 0.15   # 连败≥5: min_confidence +0.15
        elif scale <= 0.50:
            return 0.10   # 连败≥3: min_confidence +0.10
        return 0.0
```

```python
def get_strategy_kelly(self, strategy_name: str) -> float:
    """Calculate half-Kelly for a specific strategy based on recent trades."""
    rows = self._db.execute(
        "SELECT pnl FROM trades WHERE status='CLOSED' AND strategy=? "
        "AND pnl IS NOT NULL ORDER BY exit_time DESC LIMIT 20",
        (strategy_name,)
    ).fetchall()

    if len(rows) < 10:
        return 0.15  # 数据不足，用保守默认值

    pnls = [r[0] for r in rows]
    wins = [p for p in pnls if p > 0]
    losses = [abs(p) for p in pnls if p <= 0]

    if not wins or not losses:
        return 0.15

    win_rate = len(wins) / len(pnls)
    avg_win = sum(wins) / len(wins)
    avg_loss = sum(losses) / len(losses)
    payoff_ratio = avg_win / avg_loss if avg_loss > 0 else 1.0

    # Kelly = (b*p - q) / b
    kelly = (payoff_ratio * win_rate - (1 - win_rate)) / payoff_ratio
    half_kelly = kelly * self._kelly_fraction  # config.kelly_fraction = 0.5

    # Clamp: floor 0.01, cap 0.15
    return max(0.01, min(0.15, half_kelly))
```

```python

if strategy_kelly <= 0.01:
```

### 3.4 #4 Ichimoku Cloud 策略

**新文件**：`strategy/ichimoku_cloud.py`

**来源**：FreqST IchiV1 策略，简化并适配 BaseStrategy 接口。

**核心逻辑**：

```python
class IchimokuCloud(BaseStrategy):
    """Ichimoku Cloud trend strategy.

    Uses 5 components: Tenkan-sen, Kijun-sen, Senkou Span A/B, Chikou Span.
    Entry when Tenkan crosses Kijun in cloud-confirmed direction.

    Based on FreqST ichiV1 (27% ROI, 9.17 trades/day).
    """

    def generate(self, klines, symbol, regime):
        if len(klines) < 60:
            return []

        close = klines["close"]
        high = klines["high"]
        low = klines["low"]

        # Ichimoku components (ta library)
        ichi = ta.trend.IchimokuIndicator(high, low, close,
                                          window1=9, window2=26, window3=52)
        tenkan = ichi.ichimoku_conversion_line()   # 转换线 (9期)
        kijun = ichi.ichimoku_base_line()           # 基准线 (26期)
        senkou_a = ichi.ichimoku_a()                # 先行 Span A
        senkou_b = ichi.ichimoku_b()                # 先行 Span B

        # ATR for SL/TP
        atr = ta.volatility.average_true_range(high, low, close, window=14).iloc[-1]
        price = close.iloc[-1]

        # Current values
        t = tenkan.iloc[-1]
        k = kijun.iloc[-1]
        t_prev = tenkan.iloc[-2]
        k_prev = kijun.iloc[-2]
        sa = senkou_a.iloc[-1]
        sb = senkou_b.iloc[-1]
        cloud_top = max(sa, sb)
        cloud_bottom = min(sa, sb)
        cloud_thickness = abs(sa - sb) / price if price > 0 else 0

        signals = []

        # LONG: Tenkan crosses above Kijun + price above cloud
        if t > k and t_prev <= k_prev and price > cloud_top:
            # Confidence based on cloud thickness and distance from cloud
            dist_from_cloud = (price - cloud_top) / price
            base_conf = 0.40 + min(0.35, cloud_thickness * 50) + min(0.15, dist_from_cloud * 20)

            # Regime adjustment
            if regime in (MarketRegime.TRENDING_UP,):
                base_conf += 0.10
            elif regime == MarketRegime.RANGING:
                base_conf -= 0.15
            elif regime == MarketRegime.TRENDING_DOWN:
                base_conf -= 0.10

            confidence = min(0.95, max(0.30, base_conf))

            signals.append(TradeSignal(
                symbol=symbol,
                direction=Direction.LONG,
                confidence=confidence,
                entry_price=price,
                stop_loss=round(max(cloud_bottom, price - 2.0 * atr), 2),  # Cloud底或2ATR
                take_profit=round(price + 3.0 * atr, 2),
                strategy="ichimoku_cloud",
                regime=regime,
                timeframe_score=0,
            ))

        # SHORT: Tenkan crosses below Kijun + price below cloud
        elif t < k and t_prev >= k_prev and price < cloud_bottom:
            dist_from_cloud = (cloud_bottom - price) / price
            base_conf = 0.40 + min(0.35, cloud_thickness * 50) + min(0.15, dist_from_cloud * 20)

            if regime in (MarketRegime.TRENDING_DOWN,):
                base_conf += 0.10
            elif regime == MarketRegime.RANGING:
                base_conf -= 0.15
            elif regime == MarketRegime.TRENDING_UP:
                base_conf -= 0.10

            confidence = min(0.95, max(0.30, base_conf))

            signals.append(TradeSignal(
                symbol=symbol,
                direction=Direction.SHORT,
                confidence=confidence,
                entry_price=price,
                stop_loss=round(min(cloud_top, price + 2.0 * atr), 2),
                take_profit=round(price - 3.0 * atr, 2),
                strategy="ichimoku_cloud",
                regime=regime,
                timeframe_score=0,
            ))

        return signals
```

**SL 逻辑**：取云层边缘和 2×ATR 中更近的那个（保护性更强）。
**TP 逻辑**：3×ATR（R:R ≥ 1:1.5）。
**Ichimoku 独特优势**：云层是前瞻性的（投影到未来 26 期），能提前看到支撑/阻力变化。

### 3.5 #5 Enhanced BB+RSI 策略

**新文件**：`strategy/enhanced_bb_rsi.py`

**来源**：Freqtrade BbandRsi（5562% ROI）简化版，专为震荡市设计。

```python
class EnhancedBbRsi(BaseStrategy):
    """Enhanced Bollinger Bands + RSI + MACD confirmation.

    Looser conditions than mean_reversion (OR instead of AND),
    with MACD direction as false-signal filter.
    Designed for RANGING and LOW_VOLATILITY markets.
    """

    def generate(self, klines, symbol, regime):
        if len(klines) < 50:
            return []

        close = klines["close"]
        high = klines["high"]
        low = klines["low"]

        # Indicators
        bb = ta.volatility.BollingerBands(close, window=20, window_dev=2)
        upper = bb.bollinger_hband().iloc[-1]
        lower = bb.bollinger_lband().iloc[-1]
        middle = bb.bollinger_mavg().iloc[-1]
        rsi = ta.momentum.rsi(close, window=14).iloc[-1]
        macd_hist = ta.trend.macd_diff(close).iloc[-1]
        macd_hist_prev = ta.trend.macd_diff(close).iloc[-2]
        adx = ta.trend.adx(high, low, close, window=14).iloc[-1]
        atr = ta.volatility.average_true_range(high, low, close, window=14).iloc[-1]
        price = close.iloc[-1]

        # Only trade in non-trending markets
        if adx > 28:
            return []

        signals = []
        bb_width = upper - lower if upper > lower else 1.0

        # LONG: RSI oversold OR price in lower BB zone + MACD turning up
        long_rsi = rsi < 35
        long_bb = price < (lower + bb_width * 0.25)  # Lower 25% of BB
        long_macd = macd_hist > macd_hist_prev  # MACD improving

        if (long_rsi or long_bb) and long_macd:
            dist_ratio = (price - lower) / bb_width if bb_width > 0 else 0.5
            base_conf = 0.40 + min(0.40, (1.0 - dist_ratio) * 0.5)
            if long_rsi and long_bb:
                base_conf += 0.10  # Both conditions = stronger signal

            if regime == MarketRegime.RANGING:
                base_conf += 0.10
            elif regime in (MarketRegime.TRENDING_DOWN,):
                base_conf -= 0.10

            confidence = min(0.95, max(0.30, base_conf))

            signals.append(TradeSignal(
                symbol=symbol,
                direction=Direction.LONG,
                confidence=confidence,
                entry_price=price,
                stop_loss=round(price - 1.5 * atr, 2),
                take_profit=round(upper, 2),  # TP at upper BB
                strategy="enhanced_bb_rsi",
                regime=regime,
                timeframe_score=0,
            ))

        # SHORT: RSI overbought OR price in upper BB zone + MACD turning down
        short_rsi = rsi > 65
        short_bb = price > (upper - bb_width * 0.25)
        short_macd = macd_hist < macd_hist_prev

        if (short_rsi or short_bb) and short_macd:
            dist_ratio = (upper - price) / bb_width if bb_width > 0 else 0.5
            base_conf = 0.40 + min(0.40, (1.0 - dist_ratio) * 0.5)
            if short_rsi and short_bb:
                base_conf += 0.10

            if regime == MarketRegime.RANGING:
                base_conf += 0.10
            elif regime in (MarketRegime.TRENDING_UP,):
                base_conf -= 0.10

            confidence = min(0.95, max(0.30, base_conf))

            signals.append(TradeSignal(
                symbol=symbol,
                direction=Direction.SHORT,
                confidence=confidence,
                entry_price=price,
                stop_loss=round(price + 1.5 * atr, 2),
                take_profit=round(lower, 2),
                strategy="enhanced_bb_rsi",
                regime=regime,
                timeframe_score=0,
            ))

        return signals
```

```python
class MarketRegimeDetector:
    def __init__(self):
        # Per-symbol state (detect() is called with different symbols' klines)
        self._last_regime: Dict[str, MarketRegime] = {}
        self._regime_change_bar: Dict[str, int] = {}
        self._transition_cooldown: int = 6  # 6 bars (30min on 5m TF)

    def detect(self, klines, symbol: str = "") -> MarketRegime:
        # ... 现有 ADX/EMA/BB 逻辑得出 raw_regime ...

        # Transition detection
        adx_values = ta.trend.adx(high, low, close, window=14)
        adx_current = adx_values.iloc[-1]
        adx_3_ago = adx_values.iloc[-4] if len(adx_values) > 3 else adx_current

        # ADX rapid change = regime transitioning
        adx_drop = adx_3_ago - adx_current
        if adx_drop > 8:  # ADX dropped >8 in 3 bars
            return MarketRegime.TRANSITIONING

        # Momentum divergence detection
        rsi = ta.momentum.rsi(close, window=14)
        if len(close) > 10:
            price_lower = close.iloc[-1] < close.iloc[-5]
            rsi_higher = rsi.iloc[-1] > rsi.iloc[-5]
            if price_lower and rsi_higher:  # Bullish divergence
                return MarketRegime.TRANSITIONING
            price_higher = close.iloc[-1] > close.iloc[-5]
            rsi_lower = rsi.iloc[-1] < rsi.iloc[-5]
            if price_higher and rsi_lower:  # Bearish divergence
                return MarketRegime.TRANSITIONING

        # Per-symbol regime change cooldown
        last = self._last_regime.get(symbol)
        if raw_regime != last:
            self._last_regime[symbol] = raw_regime
            self._regime_change_bar[symbol] = len(klines)
            return MarketRegime.TRANSITIONING

        change_bar = self._regime_change_bar.get(symbol, 0)
        bars_since_change = len(klines) - change_bar
        if bars_since_change < self._transition_cooldown:
            return MarketRegime.TRANSITIONING

        return raw_regime

    # NOTE: detect() now accepts optional `symbol` parameter for per-symbol state.
    # Existing callers pass symbol: strategy_engine.py already has symbol available.
    # Callers that don't pass symbol will use "" as key (backward compatible).
```

### 3.7 #7 Regime 变化时自动收紧

**文件**：`defense/defense_manager.py`

**设计**：主循环在获取 `defense_result` 后，如果当前 `regime == TRANSITIONING`，覆盖参数为更保守的值。不修改 `RECOVERY_PARAMS` 模块级常量，而是在主循环中做运行时覆盖：

```python
# crypto_system.py — run_trading_cycle() 中，获取 defense_result 后
REGIME_OVERRIDES = {
    MarketRegime.TRANSITIONING: {
        "max_leverage": 5,
        "min_confidence": 0.5,
        "mtf_min_score": 5,
    },
}

if regime in REGIME_OVERRIDES:
    override = REGIME_OVERRIDES[regime]
    for key, val in override.items():
        # 取 defense_result.params 和 override 中更保守的值
        current = defense_result.params.get(key, val)
        if key == "max_leverage":
            defense_result.params[key] = min(current, val)
        elif key in ("min_confidence", "mtf_min_score"):
            defense_result.params[key] = max(current, val)
```

这样 TRANSITIONING 的保守参数与 DefenseManager 的状态机独立运作，互相取更严格的值。

```python
class MLRegimeDetector(MarketRegimeDetector):
    """ML-enhanced regime detection using LightGBM."""

    def __init__(self, model_path: str = "models/regime_model.pkl"):
        super().__init__()
        self._model = None
        if os.path.exists(model_path):
            import joblib
            self._model = joblib.load(model_path)

    def detect(self, klines) -> MarketRegime:
        if self._model is None:
            return super().detect(klines)  # Fallback to rules

        features = self._extract_features(klines)
        prediction = self._model.predict([features])[0]
        try:
            return MarketRegime(prediction)
        except ValueError:
            # Stale model may predict unknown regime, fallback to rules
            logger.debug(f"ML regime prediction '{prediction}' unknown, using rules")
            return super().detect(klines)

    def _extract_features(self, klines) -> List[float]:
        """Extract ~18 features for ML prediction."""
        close = klines["close"]
        high = klines["high"]
        low = klines["low"]
        volume = klines["volume"]

        return [
            ta.trend.adx(high, low, close, window=14).iloc[-1],
            # EMA spreads, BB width, ATR/close, RSI, volume ratio, etc.
            # ... 15-18 features total ...
        ]
```
